O que são algoritmos gulosos?

O algoritmo guloso é um paradigma de programação que procura soluções locais esperando que essas soluções também sejam soluções globais. Nem sempre essa estratégia produz uma solução ótima, porém pode produzir resultados aproximados que são satisfatórios.

O que seria uma solução local?

Ao algoritmo fazer uma escolha que no momento satisfaça as condições do problema, essa escolha deve se expandir a solução final do problema, ou seja, ao se fazer uma escolha, ela não é revista e se mantém até o final.

Exemplo de funcionamento

A estratégia gulosa, assim como o backtracking, não possui uma solução específica e pode variar de acordo com o problema. Então, tomemos o problema abaixo como exemplo.

Problema do Troco

Dado um conjunto S tipos de cédulas e um valor V , deseja-se saber qual é o número mínimo de cédulas que podemos usar para alcançar V. O problema terá como entrada um conjunto S de cédulas e um valor V. Deverá ser apresentado uma resposta com o número de cédulas de cada tipo que foi usada no troco.

Casos de teste

Entrada: S = {100, 50, 25, 10, 5, 1} e V = 275

Saída: {2, 1, 1 , 0, 0, 0}

Entrada: S = {20, 10, 5, 1} e V = 777

Saída: {38, 1, 1 , 2}

Como solucionar?

  1. C = maior cédula do conjunto;
  2. Dividir o valor V por C e isso resultará no número de cédulas de C;
  3. V = V mod C;
  4. C é retirado do conjunto de células;
  5. repete-se o passo 1 enquanto V > 0.

Dica:

Neste problema assumimos que os conjuntos de entrada vêm ordenados, porém nem todos os problemas têm essa característica, então muitos problemas são solucionados com a ordenação dos dados.

Algoritmo

#include <iostream>
#include <vector>
using namespace std;
int main(){
	int n_cedulas;
	int v;
	cin>>n_cedulas;
	int cedulas[n_cedulas];
	for(int i = 0; i < n_cedulas; ++i){
		cin>>cedulas[i];
	}	
	cin>>v;
	for(int i = 0; i < n_cedulas; ++i){
		cout<<"Cédula "<<cedulas[i]<<": "<<v/cedulas[i]<<endl;
		v  = v % cedulas[i];
	}
	return 0;
}

Aplicações

Mochila fracionária

Existe N itens em uma loja e deseja-se carregar o máximo de itens possível, porém a mochila possui uma carga máxima. Neste problema os itens podem ser fracionados.

Seleção de atividades

Uma série de atividades irão ocorrer em 1 sala e apenas uma atividade por vez pode ocorrer, então deve-se maximizar o número de atividades desta sala.
