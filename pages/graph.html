Grafos

Grafos são estruturas de dados difusas em Ciência da Computação, existem milhares de problemas computacionais que podem ser modelados utilizandos Grafos. Vale ressaltar que a teoria de grafos também é muito trabalhada na área de matemática, onde as áreas de pesquisa de computação e matemática se misturam, sendo assim, é muito comum encontar textos mais voltados para matermática que para computação relacionados a Grafos.

Mas o que é um Grafo?

Um Grafo G = (V, E) é um conjunto de vértices V e um conjunto de arestas E (de edges em inglês) utilizados para representar entidades e suas relãões. Comumente é fácil encontrar representações visuais de grafos com "bolinhas e tracinhos", onde cada vértice representa uma entidade (pessoa, cidade, etc) e cada aresta que liga dois vértices representando a relação entre os vértices. Como exemplo, imagine o Facebook e cada pessoa como um vértice, as arestas entre os vértices podem representar as amizades explicitas na rede social.

O caso do Facebook representa bem um caso de grafos não-direcionados, onde cada aresta representa uma relação dos dois lados (se A é amigo de B, B é amigo de A), mas existem também os grafos dirigídos, onde uma relação é unidirecional, por exemplo, um grafo G, onde os vértices representam cidades e as arestas as estradas ligando-as, de modo que as estradas sejam de mão única.

Representação de Grafos

Existem dois métodos comumente utilizados para representar um grafo G = (V,E) computacionalmente, como uma coleção de listas de adjacência ou por meio de uma matriz de adjacência. Cada tipo de representação tem suas vantágens, normalmente, a representação por listas de adjacência são utilizadas para grafos esparços, ou seja, que possuem poucas arestas, podendo diminuir o tempo computacional de alguns algoritmos. A representação por matriz é geralmente utilizada para grafos densos, com muitas arestas, ou quando é necessário indentificar se existe uma aresta entre dois vértices.

Aplicações

Grafos estão em todos os lugares, existem diversas aplicações que podem ser modeladas com grafos. Uma aplicação clássica é a representação de redes de comunicação, communicação de aparelhos, fluxos elétricos, fluxos hidráulicos. Geração de labirintos automaticamente para jogos, encontrar o menor caminho, como o Google Maps, simulação de moleculas químicas, todas essas aplicações podem ser modeladas utilizando a ideia de Grafos.

Busca em largura

O Percorrimento em Largura, do inglês Breadht-first Serach, é um algoritmo de percorrimento em grafos que começa a partir de algum nó, chamado de nó raiz e explora os vizinhos, antes de seguir para o póximo nível de vizinhos. O código abaixo mostra uma implementação para o BFS com uma representação dos nós como um struct e utiliza da representação da lista de adjacẽncia. 
typedef struct{
	char color = 'w';
	vector adj;
	int dist = INT_MAX;
}node;

void BFS(vector < node > &G, int n)
{
	G[n].color = 'g';
	G[n].dist = 0;
	queue< int > q;
	q.push(n);

	while(!q.empty())
	{
		int u = q.front();
		G[u].color = 'b';
		q.pop();
		for(int i = 0; i < G[u].adj.size(); ++i)
		{
			if(G[G[u].adj[i]].color == 'w')
			{
				G[G[u].adj[i]].color = 'g';
				G[G[u].adj[i]].dist = G[u].dist + 1;
				q.push(G[u].adj[i]);
			}
		}
 	}
}

Busca em profundidade

O percorrimento em profundidade, do inglês Deep-First Search, é um algortimo de perccorimento em grafos que explora o mais longe possível em cada ramo antes de buscar os vizinhos, de forma recursiva, onde para cada nó vizinho, explora-se recursivamente até o vizinho mais longe para depois retornar. 
struct Graph{
	vector edges[1000];
	bool visited[1000]; 
};

Graph g;

void dfs(Graph & g, int index){
	g.visited[index] = true;
	for(int i = 0; i < g.edges[index].size(); ++i){
		if(!g.visited[g.edges[index][i]]){
			dfs(g, g.edges[index][i], count);
		}
	}
}
